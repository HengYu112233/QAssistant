# 视频演示脚本

本文档为 QAssistant 项目的视频演示准备文件，包含演示话术、用户输入提示和预期AI回复。

## 演示前准备

- ✅ 项目已初始化（所有数据库/记忆文件已清空）
- ✅ 虚拟环境已激活
- ✅ `.env` 文件已配置（API Key 已填入）
- ✅ NapCat 已启动并监听 `ws://127.0.0.1:3001`
- ✅ 终端已进入项目根目录

## 演示流程与话术

### 段落 1：项目介绍与启动（约 30 秒）

**话术：**
"大家好，这是我们的 QQ 智能助理项目 QAssistant。它整合了 DeepSeek 大模型和 OneBot V11 协议，能够通过 QQ 群与用户进行智能对话。今天我将演示它的三个核心功能：智能对话、上下文理解和个人账务管理。让我们从启动项目开始。"

**操作步骤：**
```bash
python main.py
```

**预期输出：**
```
2026-05-11 14:30:45 - INFO - main.py - 机器人已启动，监听参数：
  QQ_WS_URL: ws://127.0.0.1:3001
  LLM_BASE_URL: https://ark.cn-beijing.volces.com/api/v3
  日志级别: INFO
[等待消息连接...]
```

**字幕标注：** "项目成功启动，现在处于监听状态"

---

"第一个功能是智能对话。我们的机器人可以理解自然语言，提供幽默而简洁的回复。让我们从一个简单的问候开始。"

**用户输入（在 QQ 群发送）：**

**预期 AI 回复：**

- 在 QQ 群中发送消息（或 NapCat 模拟器中发送）
- 机器人应在终端示出接收日志，约 1-2 秒后回复

**字幕标注：** "机器人成功理解并回复了自然语言消息"

```

### 段落 4：个人账务管理 - 记账（约 1.5 分钟）

**话术：**
"第三个功能是个人账务管理。我们在处理器中集成了账务自动识别能力。当用户提供金额和具体支出信息时，机器人会自动检测、记录并确认。这体现了我们的 MCP（模型上下文协议）能力——将外部数据库写入作为模型的可调用服务。"

**用户输入 1：**
```

```

**预期 AI 回复：**
```

总结：你今天的咖啡支出已记入账本。

**操作说明：**
- 消息会被自动解析为金额和内容
- 数据实时写入 SQLite 数据库
- 终端日志会显示 "数据库已保存"

**字幕标注：** "机器人自动识别了金额与支出项目，并已保存进数据库"

### 段落 6：架构与课程概念总结（约 1 分钟）

**话术：**
"最后，我来总结一下这个项目如何体现课程中的三个关键概念。

1. **RAG（检索增强生成）**：我们在处理器的 `_build_rag_context_messages()` 方法中，从短期对话记忆和长期记忆文件中检索上下文，并将其注入到 LLM 提示词。这使得机器人既能回忆对话历史，又能基于持久化知识生成更准确的回复。

2. **MCP（模型上下文协议）**：我们将数据库写入、QQ 回复、定时提醒等外部能力包装成可调用的服务，让 LLM 能够通过结构化消息来请求这些能力——例如识别用户的记账意图后，直接调用数据库 API 保存数据。

3. **Subagent（背景代理任务）**：在主回复完成后，机器人异步触发长期记忆更新任务。这样既不会阻塞用户的期待回复，又能在后台持续优化知识库——体现了 Subagent 独立运作的特性。"

**操作演示（可选）：**
查看终端日志，指出后台任务的输出：
```bash
# 终端应看到类似的日志输出
[后台任务] 长期记忆更新已启动 (async)
[后台任务] 提醒检查已启动 (async)
```

**字幕标注：** "项目成功整合了 RAG、MCP 和 Subagent 三大课程概念"

## 可能的补充演示（可选）

如果有多个 QQ 账号，可在不同账号中分别发送消息，演示机器人能够区分用户并维护独立的记忆和账务记录。

演示中如果网络中断或 WebSocket 连接断开，机器人会自动在 3 秒后重新连接——不需要手动重启。（可通过暂停 NapCat 再恢复来模拟）

### 代码架构导览
打开 VS Code，快速浏览：
- `main.py`：生命周期管理和守护逻辑
- `agent/processor.py`：消息处理与课程概念体现
- `core/llm_client.py`：LLM 通信与代理绕过
- `core/database.py`：SQLite 数据持久化

## 常见故障排除

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| "无法连接 WebSocket" | NapCat 未启动或端口不对 | 确认 NapCat 监听 `ws://127.0.0.1:3001` |
| "API 调用超时" | 网络延迟或代理干扰 | 检查 `.env` 中 `LLM_BASE_URL` 是否正确 |
| "机器人不回复" | API Key 错误或额度用完 | 检查 LLM_API_KEY 和账户余额 |
| "数据库锁定错误" | 多个进程访问数据库 | 确保只有一个 `main.py` 进程在运行 |
